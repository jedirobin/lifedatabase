# 反转套路库

> 用法：在故事结尾使用，打破观众预期

---

## INVERSION（反转预期）

角色行为与身份完全相反，制造反差笑点或戏剧张力。

**经典案例：**
- 幼稚的法官（在法庭上问"为什么要打架不能做好朋友"）
- 冷酷的护士（对病人毫无同情心）
- 有同理心的反派（坏人其实是好人）
- 怕黑的吸血鬼
- 不敢杀生的杀手

**使用公式：**
```
身份标签 + 相反行为 = 反转
```

**示例：**
- 职业杀手 + 养了一屋子流浪猫
- 劫匪 + 抢银行时顺便帮老人取钱
- 牧师 + 每天偷偷赌博
- 警察 + 破案靠玄学

---

## MISDIRECTION（误导揭示）

让观众以为在说A，最后揭示其实是在说B。

**经典案例：**
- CSI现场描述惨烈 → 原来是在看一张披萨
- 描述一场激烈战斗 → 原来是两只蚂蚁打架
- 描述凶杀案 → 原来是剧本杀游戏

**使用公式：**
```
严肃铺垫 + 荒诞真相 = 惊喜反转
```

**示例：**
- 描述某人被"追杀"得喘不过气 → 原来是被狗追
- 描述"最后一次见面"的悲情 → 原来只是下班告别
- 描述"背叛"的愤怒 → 原来是抢了最后一包泡面

---

## ESCALATOR（递增荒诞）

从正常的情境开始，一步步把荒诞推向极致。

**经典案例：**
- Monty Python《四个约克郡人》：比谁童年更惨，最后惨到离谱
- 《周六夜现场》：乔治华盛顿演讲，从爱国讲到"计量单位"

**使用公式：**
```
正常起点 → 第一次升级 → 第二次升级 → 彻底失控
```

**示例：**
- 主角晚睡5分钟 → 迟到 → 被开除 → 发现公司倒闭 → 发现城市毁灭 → 发现是外星人实验
- 主角捡到一块钱 → 买彩票 → 中奖 → 被绑架 → 发现绑匪是未来自己

---

## RULE OF THREE（三段式）

Setup（建立）→ Reinforce（强化）→ Subvert（打破）

**经典案例：**
- 第一次：正常
- 第二次：重复，观众预期模式
- 第三次：打破预期，制造笑点

**使用公式：**
```
第一遍：建立模式
第二遍：确认模式
第三遍：打破模式
```

**示例：**
- 三次敲门：第一次没人，第二次没人，第三次门自己开了
- 三次说"我爱你"：第一次真心，第二次敷衍，第三次对猫说
- 三次看表：第一次焦虑，第二次更焦虑，第三次发现表停了

---

## TIME LOOP（时间循环）

主角经历的事情，最后发现是循环或预演。

**经典案例：**
- 《土拨鼠之日》：每天重复同一天
- 《恐怖游轮》：死后的自己回到起点

**使用公式：**
```
事件A发生 → 角色以为结束了 → 发现回到了起点
```

**示例：**
- 主角以为逃出了密室 → 发现又回到入口
- 主角以为完成了任务 → 发现任务列表重置
- 主角以为救了人 → 发现那人又出现在危险中

---

## FALSE PROTAGONIST（假主角）

让观众以为是主角的人，其实不是。

**经典案例：**
- 《惊魂记》：以为女主是主角，结果中途被杀
- 开场跟随A角色 → A突然死亡 → 切换到B角色视角

**使用公式：**
```
建立A角色 → A做出重要决定 → A突然退场 → 真主角登场
```

**示例：**
- 跟随一个"杀手"行动 → 杀手被真正的杀手杀死
- 跟随一个"受害者"逃跑 → 受害者其实是凶手伪装的

---

## UNRELIABLE NARRATOR（不可靠叙述者）

叙述者说的和事实不符，最后揭示真相。

**经典案例：**
- 主角描述自己的"正义行为" → 画面显示其实在做坏事
- 主角说"我从不说谎" → 画面里一直在撒谎

**使用公式：**
```
叙述者说A → 画面显示B → 观众意识到被欺骗
```

**示例：**
- 主角说"我爱她" → 画面里控制欲极强
- 主角说"我是无辜的" → 闪回显示真相反转
