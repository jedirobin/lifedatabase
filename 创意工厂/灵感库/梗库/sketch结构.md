# Sketch喜剧结构

> 用法：用于创作sketch式短片，3-5分钟为宜

---

## 基础三幕结构

Sketch喜剧的核心是"喜剧前提 + 递增荒诞 + 反转收尾"

```
第一幕（SETUP）：建立前提
├── 30秒内让观众get到笑点
├── 介绍角色、场景、核心冲突
└── 抛出"喜剧前提"（What If）

第二幕（ESCALATION）：递增荒诞
├── 把前提一步步推到极致
├── 每一个"节拍"都要比上一个更夸张
└── 保持逻辑自洽，即使荒诞

第三幕（PAYOFF）：反转收尾
├── 一个意想不到的结尾
├── 可以是情绪反转、身份反转、情境反转
└── 要干脆利落，不要拖泥带水
```

---

## 喜剧前提（Comedic Premise）

好的喜剧前提要在30秒内让观众"get到"

**公式：**
```
如果 X 变成了 Y，会发生什么？
```

**示例：**
- 如果老板以为自己是超级英雄？
- 如果超市收银员把每件商品当艺术品评论？
- 如果每个人说谎头发就会变绿？
- 如果语言变成限量资源，每天只能说100字？

---

## 递增弧（Heightening Arc）

每个节拍都要比上一个更荒诞，但不能脱离设定

**示例：喜剧前提"老板以为自己是超级英雄"**

```
节拍1：老板用"超能力"开会（伸手做发射状）
节拍2：老板用"超能力"处理文件（假装读心术）
节拍3：老板用"超能力"处理危机（其实是打印机卡纸）
节拍4：老板真的"飞"出窗外（员工惊恐）
节拍5：老板...真的飞起来了？（反转：他其实是超能力者？）
```

## Sketch类型

### 1. 角色驱动型
以一个独特角色的怪癖为核心

**案例：**
- 一个过度礼貌的劫匪
- 一个相信自己是机器人的上班族
- 一个把生活当真人秀的主角

### 2. 情境驱动型
以一个荒诞情境为核心

**案例：**
- 一场只有一个人参加的考试
- 一个永远排不到你的队伍
- 一个只能问问题的面试

### 3. 社会观察型
放大日常社交行为

**案例：**
- 电梯里的尴尬沉默
- 点餐时的选择困难
- 第一次见对方家长

### 4. 荒诞型
完全脱离现实的逻辑

**案例：**
- 动物开会讨论人类
- 物品有了意识
- 时间可以买卖

### 5. 政治讽刺型
讽刺时事或社会现象

**案例：**
- 用古代背景讽刺现代问题
- 用外星人视角观察人类社会
- 夸大某个政策的影响

---

## "关于-关于"原则（The About-About）

Sketch不只是"关于"表面的情境，更深层是"关于"某个社会或人性议题

**示例：**

| 表面关于 | 深层关于 |
|---------|---------|
| 超级英雄老板 | 职场权力关系的荒谬 |
| 语言限量 | 沟通的珍贵与浪费 |
| 谎言变绿发 | 社交媒体时代的不真诚 |
| 情绪租赁 | 现代人的情感异化 |

---

## 节奏控制

- **时长**：3-5分钟最佳
- **节拍密度**：每30秒一个笑点
- **结尾**：要快，不要解释
- **信息量**：少即是多，观众只记住一个核心前提

---

## 常见错误

1. **前提太复杂**：30秒内讲不清楚
2. **没有递增**：从头到尾同一个笑点
3. **结尾拖沓**：该结束时不结束
4. **只有模仿没有创新**：光模仿形式，没有注入观点
5. **荒诞脱离逻辑**：荒诞也要有内在逻辑
