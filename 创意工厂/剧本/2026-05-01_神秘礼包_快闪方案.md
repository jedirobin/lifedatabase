## 二、完整分镜脚本（30秒版）

### 【封面层】0-3秒

| 时长 | 内容 | 画面/音效 | 备注 |
|------|------|-----------|------|
| 0-3秒 | 手机通知栏弹出"神秘礼包"推送 | 锁定手机屏幕视角，通知栏特写<br>音效：短促的"叮"提示音 | **关键帧**：前3秒留不住人，后面全是白搭 |

**文案（屏幕显示）**：
```
📩 您有一条新消息
【神秘礼包】您有一份礼包待领取...
```

### 【悬念层】3-10秒

| 时长 | 内容 | 画面/音效 | 备注 |
|------|------|-----------|------|
| 3-6秒 | 用户点击通知，跳转到"领取界面" | UI界面动画展开<br>音效：神秘的"嗖"音效 | 界面设计参考各大App开屏广告风格 |
| 6-8秒 | 出现三个礼品盒图标，标注"随机掉落" | 三个礼物图标依次出现，有轻微晃动吸引点击<br>音效：轻快的礼盒碰撞声 | 编剧小林注：这个抖动要克制，别让观众觉得在催他 |
| 8-10秒 | 弹出提示："点击领取您的神秘礼包" | 大号按钮出现，红色强调色<br>音效：心跳感的鼓点 | 这个按钮要够大够明显 |

```
🎁 点击下方按钮
   领取神秘礼包

```

### 【揭晓层】10-22秒

| 时长 | 内容 | 画面/音效 | 备注 |
|------|------|-----------|------|
| 10-12秒 | 点击后短暂黑屏+震动反馈 | 黑屏0.5秒+手机震动<br>音效："砰"的开礼音效 | 这个反转还不够痛——要够响 |

## 三、互动设计细节

### 3.1 循环抽奖机制

| 元素 | 设计说明 |
|------|----------|
| **抽奖入口** | 每个礼包揭晓后，底部出现"再来一次"按钮 |
| **循环次数** | 限制3次循环，超出后显示"今日抽奖次数已用完，明日再来" |
| **触发动作** | 点击按钮→黑屏0.5秒→重新随机抽取一个礼包 |
| **视觉反馈** | 每次循环礼盒颜色/动效有轻微变化，避免重复感 |

### 3.2 评论区引导设计

**核心话术（评论区置顶）**：
```
🎁 五一神秘礼包来袭！
   截图发评论区
   看看你抽到的是不是和我一样？
   
   #五一调休 #打工人 #职场吐槽
```

**互动话术模板**：
- "抽到调休通知的都是天选打工人"
- "老板同款黑眼圈是隐藏款，据说概率只有0.1%"
- "评论区抽3个幸运儿送我的黑眼圈（不是）"

### 3.3 截图分享设计

**截图最佳角度**：
- 设计专门的"分享卡片"样式，底部预留账号水印位置
- 卡片尺寸：9:16竖屏比例，方便直接分享到朋友圈
- 包含元素：礼包名称+小字注释+账号logo+话题标签

**分享话术**：
```
"刚抽到的，这个五一你是什么命？"
"打工人命，该来的还是来了"
"截图别删，万一能当壁纸呢"
```

## 四、视觉风格参考

### 4.1 UI界面设计方向

| 元素 | 风格建议 | 参考来源 |
|------|----------|----------|
| **整体色调** | 深色背景+霓虹渐变（紫→蓝→粉） | 抖音爆款通知风格 |
| **字体** | 粗体无衬线，字号对比强烈 | App开屏广告惯用手法 |
| **图标** | 扁平化+轻微玻璃质感 | iOS通知风格 |
| **按钮** | 圆角矩形，红色强调色，阴影立体感 | App通用CTA按钮 |

### 4.2 动效建议

| 场景 | 动效类型 | 时长 | 备注 |
|------|----------|------|------|
| 通知弹出 | 从上滑入+轻微弹性 | 0.3秒 | 参考iOS通知动画 |
| 礼盒出现 | 依次从底部升起+轻微晃动 | 0.5秒/个 | 克制，别抢镜 |
| 点击领取 | 黑屏+光效炸开 | 0.3秒 | 制造"开盲盒"惊喜感 |
| 礼包揭晓 | 礼盒翻转打开+元素飞出 | 1秒 | 要有仪式感 |
| 滑动切换 | 左右平移+轻微缩放 | 0.3秒 | 参考小红书卡片切换 |

### 4.3 色彩规范

```
主色：#FF4757（警告红-强调按钮）
辅色：#5352ED（科技蓝-背景元素）
点缀：#FFD700（金色-礼包光芒）
渐变：#A29BFE → #74B9FF → #FD79A8（神秘感）
文字：#FFFFFF（主文字）/ #A0A0A0（辅助说明）
```

## 六、风险提示

### 6.1 内容合规风险

| 风险点 | 应对方案 |
|--------|----------|
| "取消调休"涉及公司政策 | 明确标注"纯属娱乐"，增加免责声明 |
| "老板同款黑眼圈"可能被投诉 | 改为更中性的表达："加班认证勋章" |
| 抽奖机制涉及概率 | 确保"再来一次"真实有效，避免虚假宣传 |

### 6.2 传播风险

| 风险点 | 应对方案 |
|--------|----------|
| 话题敏感度过高 | 增加正面引导，如"祝大家都能真正休假" |
| 被恶意截图传播 | 视频添加账号水印，截图卡片预留水印位 |
| 竞品模仿 | 尽快发布，抢占话题首发优势 |

### 6.3 数据风险

| 风险点 | 应对方案 |
|--------|----------|
| 完播率低于预期 | 测试多个封面版本，保留数据最优的 |
| 评论区互动低 | 提前准备10-20条神评论用于互动引导 |
| 分享率低 | 优化"截图卡片"设计，降低分享门槛 |
