# 闲鱼电商套利SOP手册

**版本**：v1.0  
**创建日期**：2026年4月20日  
**目标**：建立自动化电商套利系统，实现从选品到上架的全流程自动化

---

## 一、项目概述

### 1.1 核心逻辑
```
闲鱼热销商品 → 1688/拼多多找同款 → 比价筛选 → 自动上架闲鱼
```

### 1.2 盈利模式
- **信息差套利**：低价平台采购，高价平台销售
- **目标利润率**：≥30%（扣除平台费用后）
- **风险控制**：一件代发，零库存风险

### 1.3 涉及平台
| 平台 | 角色 | 操作方式 |
|------|------|----------|
| 闲鱼 | 销售端 | 云手机APP操作 |
| 1688 | 采购端 | 浏览器/APP |
| 拼多多 | 采购端 | 浏览器/APP |

## 二、全流程SOP

### Phase 1：市场调研与选品

#### 1.1 闲鱼热销榜单抓取

**操作方式**：云手机操作闲鱼APP

**抓取内容**：
- 商品标题
- 商品价格
- 销量/浏览量
- 商品图片
- 卖家信息
- 商品描述

**抓取流程**：
```
1. 打开闲鱼APP
2. 进入"搜索" → 热门搜索/榜单
3. 按品类浏览热销商品
4. 记录商品信息
5. 截图保存
```

**数据存储**：
```
./社媒运营/闲鱼电商套利/data/
├── 热销榜单_YYYYMMDD.csv
├── 商品详情/
│   ├── 商品ID_标题/
│   │   ├── 商品信息.json
│   │   └── 商品图片/
```

#### 1.2 品类筛选标准

**优先级排序**：
1. **虚拟商品**（无物流成本，高利润）
   - 会员账号
   - 课程资料
   - 软件工具
   - 设计素材

2. **小件商品**（物流成本低）
   - 手机配件
   - 数码小物
   - 饰品
   - 文创产品

3. **信息差商品**
   - 稀缺资源
   - 限时优惠
   - 会员福利

**排除品类**：
- 大件商品（物流成本高）
- 食品饮料（保质期风险）
- 品牌仿品（侵权风险）
- 需要资质的商品

#### 1.3 利润计算公式

```
利润 = 闲鱼售价 - 采购成本 - 平台手续费 - 物流成本

平台手续费：
- 闲鱼：0%（个人卖家）
- 1688：无
- 拼多多：无

物流成本（参考）：
- 小件包邮：3-5元
- 中件包邮：8-15元
- 大件：15-30元

目标利润率 = 利润 / 采购成本 ≥ 30%
```

### Phase 2：同款比价

#### 2.1 1688同款搜索

**搜索方式**：
```
1. 提取闲鱼商品关键词
2. 在1688搜索同款
3. 使用"以图搜图"功能
4. 对比价格、评价、发货地
```

**筛选标准**：
- 起批数量：1件起批
- 发货速度：24小时内
- 店铺评分：≥4.5
- 支持一件代发

#### 2.2 拼多多同款搜索

**搜索方式**：
```
1. 提取闲鱼商品关键词
2. 在拼多多搜索同款
3. 对比价格、销量、评价
4. 筛选支持一件代发的店铺
```

### Phase 3：商品上架

#### 3.1 上架前准备

**账号准备**：
- 闲鱼账号（已实名认证）
- 芝麻信用≥600（提升信任度）
- 发布≥10条动态（提升活跃度）

**素材准备**：
- 商品主图（5张）
- 商品详情图
- 商品描述文案

#### 3.2 上架流程

**云手机操作**：
```
1. 打开闲鱼APP
2. 点击"发布" → "发布闲置"
3. 上传商品图片（首图最重要）
4. 填写商品信息：
   - 标题（30字内，含关键词）
   - 描述（详细说明）
   - 价格（略高于采购价+物流）
   - 分类
   - 发货地
   - 邮费设置
5. 发布
```

#### 3.3 标题优化公式

```
[品牌/关键词] + [核心卖点] + [规格] + [适用场景]

示例：
- 原装苹果数据线 快充充电线 1米2米 适用于iPhone全系列
- 学生党必备 台灯护眼灯 USB充电 三档调光 宿舍神器
```

#### 3.4 定价策略

```
定价 = 采购成本 × (1.5 ~ 2.0)

参考因素：
- 同行最低价
- 商品稀缺度
- 季节性需求
- 促销活动
```

### Phase 4：订单处理

#### 4.1 订单监控

**监控频率**：每天早中晚各检查一次

**监控内容**：
- 新订单通知
- 买家咨询
- 评价管理

#### 4.2 发货流程

**一件代发流程**：
```
1. 收到闲鱼订单
2. 在1688/拼多多下单
3. 填写买家收货地址
4. 支付货款
5. 获取快递单号
6. 在闲鱼填写发货信息
7. 跟踪物流状态
8. 确认收货后处理售后
```

#### 4.3 售后处理

**常见问题**：
| 问题 | 处理方案 |
|------|----------|
| 不想买了 | 同意退款，保持良好信用 |
| 商品质量问题 | 联系上家退换货 |
| 物流问题 | 跟踪物流，及时沟通 |
| 差评 | 私信沟通，协商解决 |

---

### Phase 5：数据分析与优化

#### 5.1 关键指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| 曝光量 | 商品被浏览次数 | ≥100/天 |
| 点击率 | 点击/曝光 | ≥5% |
| 转化率 | 成交/点击 | ≥10% |
| 客单价 | 平均成交金额 | ≥30元 |
| 利润率 | 利润/售价 | ≥30% |

#### 5.2 优化方向

**曝光低**：
- 优化标题关键词
- 调整发布时间（晚7-10点最佳）
- 增加商品数量

**点击低**：
- 优化首图
- 调整价格
- 优化描述开头

**转化低**：
- 优化商品描述
- 增加信任背书
- 调整定价

## 三、自动化实现方案

### 3.1 技术架构

```
┌─────────────────────────────────────────────────────────┐
│                    电商套利自动化系统                      │
├─────────────────────────────────────────────────────────┤
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌──────┐ │
│  │闲鱼抓取  │ →  │同款比价  │ →  │利润计算  │ →  │自动  │ │
│  │ (云手机) │    │ (浏览器) │    │ (Python) │    │上架  │ │
│  └─────────┘    └─────────┘    └─────────┘    └──────┘ │
│       ↓              ↓              ↓              ↓    │
│  ┌─────────────────────────────────────────────────────┐│
│  │              数据存储 (SQLite/CSV)                   ││
│  └─────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────┘
```

### 3.2 执行方式

| 环节 | 执行方式 | 频率 |
|------|----------|------|
| 热销榜单抓取 | 云手机+mobile_use | 每周1次 |
| 同款比价 | 浏览器自动化 | 需要时执行 |
| 利润计算 | Python脚本 | 实时 |
| 自动上架 | 云手机APP | 人工确认后执行 |

### 3.3 人工介入点

**必须人工确认**：
- 新品上架前
- 大额订单发货前
- 售后纠纷处理
- 价格调整

**可自动化**：
- 数据抓取
- 同款搜索
- 利润计算
- 订单提醒

## 四、风险控制

### 4.1 平台风控

**闲鱼风控规则**：
- 新账号发布频率：≤5条/天
- 相同商品：≤3条
- 敏感词：避免"代购"、"原单"等

**应对策略**：
- 控制发布频率
- 商品信息差异化
- 账号养号策略

### 4.2 资金风险

**控制措施**：
- 单品投入≤100元
- 总库存价值≤1000元
- 利润到账后再扩大规模

### 4.3 信誉风险

**维护策略**：
- 及时回复买家消息
- 发货速度≤24小时
- 售后问题优先解决
- 避免差评
