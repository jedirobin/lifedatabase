## 一、当前数据质量评估

| 维度 | 评分 | 说明 |

## 二、发现的问题

### 2.1 严重问题（必须修复）

#### 问题1：评论数量严重不足
- **现象**：每条视频仅抓取3条评论
- **期望**：每条视频至少抓取20条评论（热评优先）
- **影响**：
  - 无法分析用户情感倾向
  - 无法提取用户需求/吐槽点
  - 评论热词统计无意义
- **修复建议**：
  ```json
  "comments": {
    "count": 20,  // 至少20条
    "sort_by": "hot",  // 按热度排序
    "include_replies": true  // 包含热门回复
  }
  ```

#### 问题2：作者粉丝数全为0
- **现象**：所有视频的 `author.fans_count` 都是 0
- **实际值**：万灵缝合厂（UP主）实际粉丝数应为几十万级别
- **影响**：
  - 无法判断UP主影响力
  - 无法分析"小UP爆款"vs"大UP常规"
  - 选题策略分析受限
- **修复建议**：
  - 检查粉丝数抓取接口/选择器
  - 添加数据校验：如果粉丝数为0，标记为"抓取失败"

#### 问题3：缺少搜索关键词记录
- **现象**：`search_keyword` 字段为空字符串
- **影响**：
  - 无法追溯数据来源
  - 无法分析"不同关键词下的内容差异"
  - 后续复用数据时缺少上下文
- **修复建议**：
  ```json
  {
    "search_keyword": "明日方舟 鬼畜",  // 记录实际搜索词
    "search_source": "bilibili_search"
  }
  ```

---

### 2.2 中等问题（建议优化）

#### 问题4：弹幕时间戳精度不够
- **现象**：弹幕有 `time` 字段（秒级），但缺少弹幕类型语义化
- **建议**：
  ```json
  "danmaku": {
    "time": 30.533,
    "type": 5,  // 数字类型码
    "type_name": "顶部弹幕",  // 添加语义化字段
    "content": "生活不易，鹈鹕哈气"
  }
  ```

#### 问题5：视频标签信息单薄
- **现象**：`tags` 字段只有1-2个标签
- **建议**：抓取完整标签列表（通常B站视频有5-10个标签）

#### 问题6：缺少视频封面下载
- **现象**：只有 `cover_url` 链接
- **建议**：下载封面图到本地，便于后续分析封面设计

---

### 2.3 轻微问题（可选优化）

#### 问题7：发布时间格式不友好
- **现象**：`publish_time` 是Unix时间戳
- **建议**：同时提供可读格式
  ```json
  "publish_time": 1776656700,
  "publish_time_str": "2026-04-20 14:45:00"
  ```

#### 问题8：缺少视频分区信息
- **建议**：添加 `category` 字段（如：动画、游戏、科技）
