## 三、文档同步方案

### 3.1 同步原则

| 原则 | 说明 |
|------|------|
| **单向同步** | Coze → Obsidian（Coze生成，Obsidian存档） |
| **版本控制** | 保留历史版本，支持回溯 |
| **分类归档** | 按「素材→整理→产出」三层流转 |
| **元数据标记** | 添加标签、日期、来源便于检索 |

### 3.2 同步工作流

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Coze 文档生成                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐               │
│  │ 脚本撰写 │→│ 评审优化 │→│ 内容产出 │→│ 数据归档 │               │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘               │
└───────┼───────────┼───────────┼───────────┼────────────────────────┘
        │           │           │           │
        ▼           ▼           ▼           ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        同步触发点                                     │
│  • 任务完成自动同步    • 手动触发同步        • 定时批量同步           │
└───────┬───────────┬───────────┬───────────┬────────────────────────┘
        │           │           │           │
        ▼           ▼           ▼           ▼
┌─────────────────────────────────────────────────────────────────────┐
│                      Obsidian 知识库                                 │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐               │
│  │ sources │→│ sources │→│   wiki  │→│ outputs │               │
│  │  (素材) │  │ (整理)  │  │ (编译)  │  │ (产出)  │               │
│  └─────────┘  └─────────┘  └─────────┘  └─────────┘               │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.3 同步分类规则

| Coze文档类型 | Obsidian目标目录 | 说明 |
|--------------|------------------|------|
| 脚本文件 (.md) | `wiki/项目/钟钟AIZz/内容规划/` | 经评审定稿的脚本 |
| 评审报告 | `memory/reviews/` | 季度分类归档 |
| 行业分析 | `sources/documents/行业报告/` | 原始参考资料 |
| 工具教程 | `wiki/工具/AI工具/` 或 `wiki/工具/剪辑软件/` | 工具知识库 |
| 热点追踪 | `sources/social/热点追踪/` | 原始素材 |
| 爆款分析 | `wiki/平台/B站/头部博主/` 或 `小红书/头部博主/` | 整理后入库 |
| 数据采集脚本 | `scripts/爬虫/` | Python脚本归档 |
| 评审脚本 | `scripts/评审/` | 自动化脚本 |

### 3.4 同步执行方案

#### 方案A：Git同步（推荐）

```bash
# 1. Coze端：配置Git仓库
cd ~/coze_workspace
git init
git remote add origin https://github.com/yourname/lifedatabase-sync.git

# 2. 创建同步脚本 sync_to_obsidian.sh
#!/bin/bash
# Coze工作目录
COZE_DIR="."
# Obsidian仓库
OBSIDIAN_DIR="E:/obsidianbase/lifedatabase"

# 同步社媒运营文档
rsync -av --include='社媒运营/**' --exclude='*' "$COZE_DIR/" "$OBSIDIAN_DIR/"

# 同步基础文档
rsync -av --include='USER.md' --include='MEMORY.md' --exclude='*.md' "$COZE_DIR/" "$OBSIDIAN_DIR/"

# 同步脚本
rsync -av --include='*.py' --exclude='*' "$COZE_DIR/" "$OBSIDIAN_DIR/scripts/"

# 提交并推送
cd "$OBSIDIAN_DIR"
git add -A
git commit -m "Coze sync: $(date '+%Y-%m-%d %H:%M')"
git push origin main
```

#### 方案B：云盘同步（备选）

```markdown
使用OneDrive/Dropbox/坚果云等云盘服务：

1. Coze生成文档保存到云盘同步目录
2. Obsidian知识库也放在同一云盘目录
3. 自动同步，无需手动操作

适合场景：

```

## 四、快速开始指南

### 4.1 首次设置

```bash
# 1. 克隆Obsidian仓库到本地
git clone https://github.com/yourname/lifedatabase.git ~/lifedatabase

# 2. 关联Obsidian仓库
# Obsidian → 设置 → 插件 → 启用 "Obsidian Git"
# 配置自动提交间隔：5分钟

# 3. 配置同步脚本
# 将 sync_to_obsidian.sh 放到Coze工作目录

# 4. 初始化Dataview索引
# 在Obsidian中运行 Dataview: Rebuild Index
```

### 4.2 日常使用流程

```
1. Coze生成内容
   ↓
2. 内容保存到工作目录
   ↓
3. 运行同步脚本（或等待自动同步）
   ↓
4. Obsidian自动更新
   ↓
5. 后续在Obsidian中继续编辑/整理
```

### 4.3 冲突处理

当Coze和Obsidian同时修改同一文件时：

| 策略 | 说明 |
|------|------|
| **Coze优先** | 社媒运营内容以Coze为源，Obsidian作为备份 |
| **手动合并** | 重要文档发生冲突时，人工判断合并 |
| **版本保留** | 冲突时保留两个版本，标记`.conflict`后缀 |
